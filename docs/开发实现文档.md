## 三、功能一：对话式学习画像自主构建

### 3.1 功能目标

用户通过自然对话，系统自动从对话中提取特征，构建包含6个维度的学习画像。

### 3.2 六维度画像定义

| 维度 | 说明 | 取值示例 | 置信度 |
|:---|:---|:---|:---|
| **知识基础** | 对当前学习领域的掌握程度 | 初学/入门/进阶/精通 | 0.0-1.0 |
| **认知风格** | 偏好的信息接收方式 | 视觉型/听觉型/动觉型/阅读型 | 0.0-1.0 |
| **易错点偏好** | 常犯的错误类型或知识盲区 | 概念混淆/计算粗心/逻辑不清/记忆困难 | 0.0-1.0 |
| **学习偏好** | 喜欢的学习材料形式 | 视频为主/文字为主/练习为主/混合型 | 0.0-1.0 |
| **学习节奏** | 学习速度和耐力 | 快节奏/适中/慢节奏/碎片化 | 0.0-1.0 |
| **目标导向** | 学习目的 | 应试取证/技能实践/兴趣探索/学术研究 | 0.0-1.0 |

### 3.3 实现方案

- **ReAct推理模式**：Thought → Action → Observation 循环
- **意图识别**：自动识别用户是学习、做题、规划还是求助
- **特征提取**：从对话中提取学习特征，更新画像
- **直接回复**：不再反复追问，直接给出有用回复

## 四、功能二：多智能体协同的资源生成

### 4.1 功能目标

系统根据用户需求和画像，由多智能体协作生成至少6种类型的个性化学习资源。

### 4.2 智能体角色定义

| 智能体名称 | 核心职责 | 输出类型 |
|:---|:---|:---|
| **主管智能体** | 理解意图、拆解任务、路由调度 | 任务分配指令 |
| **RAG检索智能体** | 基于用户需求检索知识库 | 相关文档片段 |
| **资料生成智能体** | 生成专业课程讲解文档 | Markdown文档 |
| **思维导图智能体** | 将知识点转化为树状结构 | Markmap语法 |
| **习题生成智能体** | 生成不同难度的练习题 | 结构化JSON题目 |
| **代码案例智能体** | 生成可运行的实操代码 | 代码+注释 |
| **视频脚本智能体** | 生成教学视频脚本 | 脚本文本 |
| **TTS脚本智能体** | 生成知识点音频讲解 | 脚本文本 |

### 4.3 执行顺序

```
Supervisor → RAG检索 → 资料生成 → [导图||习题||代码||视频||TTS] → 质量检查 → 结果聚合
```

## 五、功能三：个性化学习路径规划和资源推送

### 5.1 功能目标

根据用户画像和学习目标，自动规划学习步骤顺序，按顺序推送个性化资源。

### 5.2 Plan-and-Execute模式

1. **Planner**：结合用户画像 + 知识库生成学习计划（5-8个步骤）
2. **Executor**：逐步执行计划，匹配/生成资源
3. **Feedback**：用户反馈"太难/太简单"，动态调整路径
4. **Re-Planner**：必要时重新规划

### 5.3 反馈处理

| 反馈类型 | 调整策略 |
|:---|:---|
| 太难 | 在当前步骤前插入前置知识步骤 |
| 太简单 | 跳过当前步骤，合并后续相似步骤 |
| 需要帮助 | 触发智能辅导，提供答疑 |
| 完成 | 进入下一步骤 |

## 七、环境配置

### 7.1 后端环境变量

```bash
# LLM 配置
LLM_PROVIDER=openai
LLM_API_KEY=sk-xxx
LLM_MODEL=deepseek-chat
LLM_BASE_URL=https://api.deepseek.com

# MongoDB
MONGODB_URL=mongodb://localhost:27017
MONGODB_DB_NAME=a3_learning_agent
```

### 7.2 启动步骤

```bash
# 1. 启动MongoDB
docker run -d --name mongodb -p 27017:27017 -v mongodb_data:/data/db mongo:7.0

# 2. 启动后端
cd backend
pip install -r requirements.txt
python run.py

# 3. 启动前端
cd frontend
npm install
npm run dev
```
