## 二、技术选型决策

| 技术 | 选型理由 |
|:---|:---|
| LangGraph | 支持循环图、持久化检查点、Human-in-the-Loop |
| FastAPI | 异步支持好，自带Swagger文档 |
| Vue 3 + Vite | 组合式API，开发体验好 |
| MongoDB | 文档型数据库，Schema灵活 |
| DeepSeek | 性价比高，中文能力强 |

## 三、核心数据流

### 3.1 对话式画像构建

```
用户输入消息
    │
    ▼
[FastAPI 接收 POST /api/chat]
    │
    ├──→ 存储消息到 MongoDB
    │
    ├──→ 启动 ProfileAgent 工作流
    │        │
    │        ├── 提取特征（ReAct循环）
    │        │
    │        ├── 更新画像
    │        │
    │        └── 生成回复
    │
    └──→ 返回 SSE流式响应
```

### 3.2 多智能体资源生成

```
用户输入学习需求
    │
    ▼
[FastAPI 接收 POST /api/generate]
    │
    ├──→ 返回 task_id
    │
    └──→ 异步启动 ResourceAgent
             │
             ├── RAG检索
             ├── 资料生成
             ├── 并行生成6种资源
             ├── 质量检查
             └── 结果聚合
```

### 3.3 学习路径规划

```
用户触发路径规划
    │
    ▼
[FastAPI 接收 POST /api/path/plan]
    │
    └──→ 启动 PathAgent
             │
             ├── Planner生成计划
             ├── Executor执行步骤
             ├── 进度追踪
             └── 反馈处理
```

## 四、部署方案

### 4.1 开发环境

```bash
# MongoDB
docker run -d --name mongodb -p 27017:27017 mongo:7.0

# 后端
cd backend && python run.py

# 前端
cd frontend && npm run dev
```

### 4.2 生产环境

- 前端：Vercel / Netlify 部署
- 后端：Docker Compose 部署
- 数据库：MongoDB Atlas 云服务
